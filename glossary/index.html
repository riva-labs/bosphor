Glossary

Key terms used throughout the Bosphor documentation.

ABCDEFGHIJKLMNOPQRSTUVWXYZ

B

Blob

A unit of data stored on Walrus. Bosphor stores intent payloads as deletable blobs.

C

Cross-chain message

A message sent between chains via LayerZero v2.

D

DVN (Decentralized Verifier Network)

A set of independent verifiers in LayerZero v2 that validate cross-chain messages before delivery.

E

EID (Endpoint ID)

A numeric identifier for a chain in LayerZero v2.

Epoch

A time period in Walrus that determines how long a blob is stored.

I

Intent

A storage request submitted on an EVM chain containing payload, deadline, and destination chain EID.

L

LayerZero v2 (LayerZero, LZ)

The cross-chain messaging protocol Bosphor uses for both directions of the message flow.

O

OApp (Omnichain Application)

A LayerZero v2 application contract that can send and receive cross-chain messages.

P

Peer

A cross-chain OApp counterpart configured via setPeer (EVM) or set_peer (Sui).

Proof

An execution receipt sent from Sui back to EVM after a storage intent is fulfilled.

R

Relayer

The off-chain service that processes intents: polls events, uploads to Walrus, sends proof back to EVM.
